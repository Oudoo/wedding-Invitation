Bootstrapping Projects

Initialize your environment correctly to ensure agents obey the governance rules from the first commit.

New Project Automation
new <project-name>

When: Starting a fresh project from scratch.

How: Clones template, creates private repo, generates a fresh README, and scaffolds .agent/HANDOFF.md.

"Saves 10 minutes of manual setup and ensures 100% compliance."

Retrofit Governance
agent-os

When: Adding rules to an existing repository.

How: Injects .agent/AGENTS.md, proxies, and appends to .gitignore.

"Prevents agents from hallucinating legacy folder structures."

The Governance Core

Bootstrapping installs the .agent/AGENTS.md file. This is the "brain" of your project. It contains strict directives that forbid committing to main and force agents to sync before starting.

Branching & Isolation

Manage parallel work streams and multi-app repositories without file collisions.

Subproject
branch <name>

When: Starting a new app or service inside an umbrella repo.

How: Creates an empty orphan branch (no history), seeds .agent/HANDOFF.md, and immediately publishes to GitHub.

"Treat an empty branch as a completely new, isolated project."

Parallelism
worktree <branch>

When: Running multiple agents (e.g. Claude + Antigravity) at once.

How: Spawns an isolated physical directory for that specific branch.

"Prevents agents from breaking each other's dev servers."

Development Optimization

Reduce token consumption by up to 40% using structural mapping and surgical rules.

The 'Map' Directive

Always run map before launching an agent. This builds a compressed text tree at .agent/repo-map.md, stopping the agent from reading every file just to understand the folder structure.

map

1. Surgical Read

Agents are forbidden from reading files over 200 lines if they only need specific blocks. They must use targeted grep or line ranges.

2. No Probing

Dependencies and payloads are pinned in code comments so agents don't waste tokens "searching" for API versions online.

3. Memory Reuse

Agents are forced to rely on internal session memory for files they already modified in the same turn.

The Handoff Protocol

The bridge between agents. GitHub holds the code; .agent/HANDOFF.md holds the memory.

Phase A: The Outbound (Leaving Agent)

Paste this to the agent you are currently using when tokens are low or you hit a hard blocker.

We are pausing this session and handing off to another agent. 
Perform our standard handoff protocol immediately:
1. Stage and commit all active files.
2. Push this branch to origin.
3. Update the `.agent/HANDOFF.md` file using the machine-readable template. Fill in LAST_AGENT, BRANCH, LAST_COMMIT, UPDATED, GOAL, CURRENT_STATE, BLOCKER, NEXT_STEP, and FILES.
Be concise. Do not include any conversational summary. Execute now.

Phase B: The Inbound (Arriving Agent)

Paste this to the new agent you just opened in the repository.

We are resuming a development session. Execute these steps:
1. Run `git pull` to fetch the latest codebase state.
2. Read `.agent/HANDOFF.md` to capture our goal, state, and blockers.
3. Load the listed FILES into your context.
4. Execute the 'NEXT_STEP' listed in that file.
Do not give me any introductory conversational filler. Go straight to coding.

State Recovery

Use checkpoints for micro-pauses or unexpected task switches without writing a full handoff note.

The 'Checkpoint' Command

Freezes local uncommitted work with a 1-line hint.

checkpoint "middle of debugging odoo payload"

Injects timestamp/message, auto-commits, and safely pushes only the current branch (origin HEAD).

The 'Resume' Command

Instantly restores your mental context before launching an agent.

resume
🔄 Resuming Agent Session...
🎯 GOAL: Build OAuth flow
🛑 BLOCKER: 401 invalid redirect URI
⏭️ NEXT STEP: Fix redirect URI generation

System Health

Verify your repository is AgentOS compliant before initiating heavy autonomous tasks.

The 'Doctor' Command

When working with dozens of repositories, you need to ensure the environment is pristine. The doctor command runs a comprehensive check on Git state, remotes, and the presence of critical AgentOS governance files.

doctor

  ✓ Git initialized
  ✓ Remote connected

  ✓ CLAUDE.md exists
